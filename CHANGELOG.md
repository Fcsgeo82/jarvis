## [2.0.0] — 2026-06-17

### Added

#### Voice Agent (Triple Clap / Wake Word)
- **Triple-clap pattern** (3 claps within 1 second) triggers a full voice interaction loop
- **faster-whisper** integration for 100% offline speech-to-text (no API key required)
  - Configurable model via `WHISPER_MODEL` env var (tiny → large-v3)
  - Records 5 seconds of audio via `sounddevice`
- **DeepSeek-V3 LLM** integration via `openai` client (OpenAI-compatible API)
  - System prompt in Portuguese; concise responses (max 3 sentences)
  - Configurable via `DEEPSEEK_MODEL` env var (default: `deepseek-chat`)
  - Supports `deepseek-reasoner` (R1) for advanced reasoning tasks
- **Conversation memory** persisted to `.cache/jarvis_memory.json`
  - Rolling window of last 20 messages (configurable via `MAX_HISTORY_MESSAGES`)

#### Tool Calling / Function Calling
- **`get_weather(city)`** — real-time weather via wttr.in; no API key required
- **`search_web(query)`** — DuckDuckGo web search via `ddgs`; no API key required
- **`set_reminder(seconds, message)`** — schedules a spoken reminder via `threading.Timer` + ElevenLabs TTS
- Full two-pass function calling loop: DeepSeek → tool execution → DeepSeek final reply

#### Dynamic Welcome (Feature 7)
- Boot sequence welcome phrase now generated by DeepSeek at runtime
  - Includes current time, day of week, and a motivational tip
  - Falls back to static English phrase if `DEEPSEEK_API_KEY` is not set
- Static fallback: `"Welcome home sir. All systems are ready and waiting your command."`

#### Wake Word — Offline Activation (Feature 6)
- **openwakeword** integration running in a dedicated daemon thread
- Detects `"hey jarvis"` using ONNX model (downloaded automatically on first run, ~5 MB)
- 3-second cooldown between wake word triggers to prevent double-firing
- Fully parallel to the clap-detection main loop

#### System Tray Icon (Feature 8)
- **pystray** + **Pillow** system tray icon with live status feedback
  - 🟢 Green circle — listening
  - 🟡 Yellow circle — processing / recording
  - 🔵 Blue circle — speaking
- Tooltip updates with current action description
- Tray menu: status label + **Sair** (quit) option

#### Antigravity IDE Integration
- Replaced all Cursor IDE logic with **Antigravity IDE**
- Auto-detects `Antigravity IDE.exe` via registry-style path lookup
- `FOCUS_EXISTING_ANTIGRAVITY_ON_DOUBLE_CLAP = True`: brings existing window to foreground
- Sends F11 to toggle fullscreen after focus
- Override exe path via `ANTIGRAVITY_EXE` env var

### Changed
- `jarvis.py` fully rewritten to support dual-mode clap detection (double vs. triple)
- `requirements.txt` expanded with 6 new dependencies
- `.env.example` updated with all new variables and Portuguese comments
- `_ask_deepseek` replaced by `_ask_deepseek_with_tools` supporting the full function-calling loop
- `say_jarvis_welcome` now calls `generate_dynamic_welcome()` instead of using a hardcoded phrase
- Clap detection refactored to track a rolling `clap_times` list for multi-clap pattern matching

### Fixed
- **Windows UTF-8 encoding**: `sys.stdout.reconfigure(encoding='utf-8', errors='replace')` added at startup — prevents `UnicodeEncodeError` when logging emoji-containing weather responses (wttr.in)
- **`duckduckgo_search` → `ddgs`**: Migrated to the new package name with a graceful fallback import for both package names

### Dependencies Added

| `openai` | `>=1.0` | DeepSeek API client |
| `faster-whisper` | `>=1.0` | Offline STT |

## [1.0.0] — 2025 (original)

### Initial Release

- **Double-clap detection** via microphone using `sounddevice` + `numpy`
  - Adaptive noise floor with exponential moving average
  - Configurable `SPIKE_RATIO`, `COOLDOWN_S`, `MIN_RMS`, `BLOCK_MS`
- **Boot sequence** on double clap:
  - Opens Spotify track via URI (`os.startfile`)
  - Opens Claude in Chrome (Monitor 1, fullscreen)
  - Opens Binance BTC chart in Chrome (Monitor 3, fullscreen)
  - Focuses Cursor IDE (Win32 `EnumWindows` + `SetForegroundWindow`)
  - Speaks welcome phrase via ElevenLabs TTS
- **ElevenLabs TTS** with PCM audio playback via `sounddevice`
  - WAV caching to avoid repeat API calls for the same phrase
  - Configurable voice, model, and output format
- **Multi-monitor Chrome placement** via Win32 `EnumDisplayMonitors`
  - F11 fullscreen toggle via `keybd_event`
  - Per-site Chrome profiles (`--user-data-dir`)
- Environment variable loading via `python-dotenv`
